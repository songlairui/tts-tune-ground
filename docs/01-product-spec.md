# Spec: TTS API 调试台

## 核心目标

100% 覆盖小米 MIMO TTS API 的所有能力，不遗漏任何参数、任何模式、任何功能。让开发者可以通过界面完整探索 API 的边界。

**定位：** API 能力探索工具。先覆盖，再优化。

**目标用户：** 需要深度探索 TTS API 边界的开发者和创作者。

---

## 已知 API 能力（已验证）

### 三种模型模式

| 模型 ID | 核心能力 | 音色来源 |
|---------|----------|----------|
| `mimo-v2.5-tts` | 基础合成 + 唱歌模式 | 8 种内置音色 |
| `mimo-v2.5-tts-voicedesign` | 文字描述设计音色 | 音色描述文本 |
| `mimo-v2.5-tts-voiceclone` | 音频克隆音色 | 上传音频 base64 |

### 两种风格控制方式

1. **自然语言控制**（user message）：自由描述，支持导演模式三段式
2. **标签控制**（assistant message）：`(开头整体风格)` + `[内嵌细粒度标签]`

### 音频参数

- 格式：`wav`（非流式）、`pcm16`（流式）
- 流模式：`stream: true/false`（注意：真实流式暂不可用，兼容模式）

### API 限制

- 单次最长文本：约 5000 字（用户实测）
- 克隆音频：≤ 10MB，支持 mp3/wav
- 上下文窗口：8K

---

## 已解答的开放问题

✅ API Key 通过 `api-key` header 传递，不是 Authorization Bearer  
✅ 内置音色共 8 种，4 中 4 英  
✅ 流式输出目前是兼容模式（全部生成完才返回）  
✅ 不支持 SSML，用自定义标签系统  
✅ 限率：暂不考虑，遇到再处理

#### 内置音色 Tab
- 8 种音色下拉选择，标注语言/性别
- 预览按钮播放该音色采样

#### 音色设计 Tab
- 多行描述文本框
- 示例 prompt 快速填入（ASMR/播客/纪录片/游戏）

#### 音色克隆 Tab
- 拖拽上传 mp3/wav，≤ 10MB
- 波形预览，自动转 base64 data URI

#### 标签插入工具栏
按分类组织所有可用标签，点击即插入光标位置：
- **基础情绪**：开心/悲伤/愤怒/恐惧/兴奋/平静
- **复杂情绪**：忧郁/释然/无助/愧疚/疲惫
- **节奏控制**：吸气/深呼吸/叹气/喘息/屏住/停顿
- **声音特征**：颤抖/破音/鼻音/气声/沙哑
- **表情语气**：微笑/轻笑/大笑/冷笑/抽泣/哽咽
- **方言**：东北话/四川话/粤语
- **唱歌模式**：在文本开头插入 `(唱歌)` 标签

### Phase 2: 缓存与历史 - "不重复生成同样的东西"

**目标：** 节省 tokens，快速回溯之前的尝试。

1. **自动本地缓存**
   - 相同参数 + 相同文本 = 直接从 IndexedDB 返回
   - 缓存命中时显示「缓存命中，0 token」提示
   - 可手动清除缓存

2. **历史记录侧边栏**
   - 最近 50 条生成记录，时间倒序
   - 每条显示：微型波形缩略图 + 前 20 字 + 音色名
   - 点击任意历史 → 恢复所有参数 + 音频
   - hover 自动播放前 2 秒预览

3. **预设系统**
   - 保存当前所有参数为命名预设
   - 预设分类：工作用 / 试错中 / 已定稿
   - 从预设一键加载

---

### Phase 3: 长文本模式 - "5000 字分段处理"

**目标：** 长文本不是一次性生成，而是逐段打磨

自动触发：文本超过 500 字，或手动点击「文档模式」

#### 段落级操作

```
┌─────────────────────────────────────────────┐
│ §1 (轻快)在一个阳光明媚的早晨...            │ ✅ 🔊
│                                             │
│ §2 他推开门，[深吸一口气]眼前的景象...      │ ✏️ 🔊
│                                             │
│ §3 这里是第三段，还没生成...                │ 🚀 🔊
│                                             │
│ [ 生成全部未生成 ]  [ 播放全文 ]            │
└─────────────────────────────────────────────┘
```

- 自动分段（按空行/标点），支持手动合并/拆分
- 全局风格 + 单段覆盖：每一段可以单独设置风格指令
- 单独生成某一段，不影响其他
- 段落拖拽重排，音频自动拼接
- 断点续传：刷新后已生成段落从缓存恢复

#### 关键探索点：音色一致性

> **问题：** 多段生成时，如何保持音色和语气的一致性？

**调试台需要提供的工具：**
- 「锁定音色」开关：开启后，所有段落使用完全相同的 voice 参数 + 相同的全局风格前缀
- 「音色锚点」功能：用第一段生成的结果作为锚点，后续所有段落自动带上相同的风格描述后缀
- 「一致性检查」：对比相邻段落的声纹特征（前端简单算法），给出一致性评分

---

### Phase 4: API 边界探索 - "测试标签的想象力极限"

**目标：** 系统探索 API 文档里没写的能力边界

#### 探索功能 A：标签组合实验

> 灵感来源：用户发现可以通过标签实现"在山洞中，带有轻微回声"这种文档没写的效果

- **标签实验室**：自由组合多个标签，观察叠加效果
  ```
  (山洞回声 空旷 混响)文本内容
  ```
- 预设标签组合库：
  - 空间类：山洞/山谷/小房间/大厅/浴室回声
  - 距离类：耳边低语/远处呼喊/隔壁房间/电话听筒
  - 设备类：收音机/老唱片/对讲机/电话音
- 「随机组合」按钮：随机生成 3 个标签的组合，用于发现意外效果
- 「有效标签」收藏：用户验证有效的自定义标签，本地保存

#### 探索功能 B：A/B 参数对比

- 界面分裂为左右两个独立工作区
- 拖放两条历史到左右，只改一个参数
- 一键对比播放，听出细微差别
- 自动高亮两个请求的 JSON diff

#### 探索功能 C：批量矩阵测试

- 定义变量维度：比如 3 种风格 × 4 种音色
- 自动生成所有组合，排队生成
- 生成完成后进入评审模式：逐个播放，滑动筛选
- 输出对比报告：哪些参数组合效果最好

## 交互原则（简化版）

### 只保留三个核心快捷键

| 快捷键 | 功能 |
|--------|------|
| `Cmd + Enter` | 生成音频 |
| `Space` | 播放/暂停（光标不在文本框时） |
| `Esc` | 取消生成 / 关闭弹窗 |

### 其他微交互

- 生成按钮状态：闲置→蓝色，生成中→进度条+取消，成功→绿色闪烁，失败→红色抖动
- 新生成的历史条目从顶部滑入高亮
- JSON 预览支持一键复制全部
